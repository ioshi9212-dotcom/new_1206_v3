# Scene Output Style Layer 1206

Пакет правил для того, как GPT должен писать сцену 1206: шапка, тело сцены, диалоги, нижний блок, POV, автономия персонажей и контроль качества.

## Состав

```yaml
gpt/scene_output/
  scene_writing_format_1206_ru.md
  scene_footer_rules_1206_ru.md
  pov_context_loading_rules_1206.json
  character_autonomy_scene_rules_1206_ru.md
  scene_quality_control_1206_ru.md
```

## Главное

- Тело сцены пишется в третьем лице, но через ограниченный POV.
- Нижний блок пишется от первого лица текущего POV.
- Не писать: `Акира может сделать`.
- Писать: `Что я могу сделать`.
- Мысли — внутренним голосом POV, не `Акира думает`.
- Реплики и действия в footer — максимум по 3.
- Отношения — только актуальные, максимум 3 строки.
- Состояние — только состояние.
- Акира не грузится автоматически, если POV не Акира и её нет в сцене.
- Персонажи не живут прицепом к Акире.
- Вопрос NPC может перебить запланированный уход, и тогда сцену надо остановить на выборе.

## Куда подключать

Эти файлы логично подключать в финальный prompt / runtime scene rules digest после того, как сборщик `scene_contract` начнёт тянуть:

- активный POV;
- активных персонажей;
- текущую локацию;
- character_memory;
- relationship_pairs;
- NPC presence rules;
- calendar slice.

Промт лучше править после сборщика, чтобы GPT получал чистый контекст и не держал всё на словах.
