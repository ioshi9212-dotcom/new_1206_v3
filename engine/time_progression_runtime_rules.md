# Time Progression Runtime Rules

## Главное

Время сцены = смысловая длительность событий, а не количество сообщений.

Один ход не равен одной минуте.

## Перед шапкой сцены

Перед написанием шапки оценить:

1. Какое время было в `current_state.time`.
2. Что сделал игрок.
3. Что сделали NPC.
4. Были ли паузы, взгляды, перемещение, ожидание, сборы, разговор, конфликт.
5. Сколько минут это заняло по смыслу.
6. Какое новое время должно быть в шапке.
7. Какое время нужно сохранить через `applyTurnResult`.

## Типовые сдвиги

- мгновенная реакция, один взгляд: 0–1 минута;
- короткий обмен 1–2 реплики: 1–2 минуты;
- обычный диалог с паузами: 3–7 минут;
- напряжённый спор/допрос/конфликт: 5–15 минут;
- слушать за дверью несколько реплик: 2–6 минут;
- собрать вещи, взять записку, проверить окно: 3–10 минут;
- путь внутри дома/малой зоны: 1–5 минут;
- путь между зонами базы: 5–15 минут;
- еда/кофе: 10–25 минут;
- переодевание/душ/сборы: 10–30 минут;
- медосмотр: 10–40 минут;
- тренировка: 20–90 минут;
- сон/таймскип: до указанного игроком или следующего важного beat.

## Нельзя

- двигать время механически на 1 минуту каждый ход;
- оставлять то же время после нескольких действий;
- писать длинный диалог на 6–10 реплик, где прошло только 1 минута;
- считать технические ходы игровым временем;
- менять дату без таймскипа или календарной причины.

## Обязательное сохранение

После игровой сцены `applyTurnResult` должен обновить:

```json
{
  "current_state_patch": {
    "time": "HH:MM",
    "last_time_advance_min": 0,
    "last_time_advance_reason": "почему прошло столько времени"
  }
}
```

Если время не изменилось, причина должна быть: мгновенная реакция / продолжение того же момента.
