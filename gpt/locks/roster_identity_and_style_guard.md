# Roster, identity and style guard — 1206

Этот файл задаёт системные правила сборки сцены. Он не является текстом сцены и не содержит готовых реплик.

## Roster continuity

Текущий roster берётся из физически подтверждённого кадра: кто находится рядом, говорит, смотрит, удерживает маршрут, блокирует проход или влияет на ближайший выбор Акиры.

Если `current_state` противоречит последнему видимому кадру, приоритет у последнего видимого кадра и scene history. Старый стартовый roster не должен оставаться активным после смены локации.

Персонаж, который физически находится в сцене и влияет на действие, должен быть в active/nearby roster до рендера.

## Identity binding

Видимый дескриптор и backend character_id — разные уровни.

Если сцена использует фамилию, должность, описание одежды, транспорт, телесный маркер или другую видимую метку уже известного персонажа, runtime должен привязать это к его character_id для загрузки карточки. Видимый текст при этом продолжает соблюдать POV и known-name rules.

Фамилия или видимый дескриптор не создают нового NPC, если в character index уже есть подходящий персонаж.

## Known-name timing

Знание имени проверяется на момент строки, а не задним числом.

Персонаж может использовать имя только если до этой строки у него был источник: услышал в сцене, прочитал, знал раньше или получил через state/документ. Более поздняя реплика не оправдывает более раннее употребление имени.

Если источник имени есть только у backend/runtime, но не у POV или speaking NPC, в видимом тексте остаётся дескриптор, фамилия или нейтральное обращение по доступному знанию.

## Character files

Если персонаж активен или физически рядом, fast context должен подтянуть его основные файлы: `main.yaml`, `character.yaml`, `knowledge.yaml`.

`past.yaml` не является default-файлом поведения. Он подключается только при сценическом триггере прошлого, памяти, Самуэля, лабораторий, Эхо, шрамов, кольца или прямого раскрытия старой связи.

## Paragraph flow

Сцена не должна по умолчанию идти вертикальным столбиком из коротких обрывков.

Обычная проза собирает связанный видимый beat в читаемый абзац: положение, движение, реакция, последствие, давление. Отдельная короткая строка нужна только для реплики, резкого действия, смены ритма или сильной остановки.

Если несколько коротких фраз описывают один и тот же кадр без новой информации, их нужно объединить или убрать.
