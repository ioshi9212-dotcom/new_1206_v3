# 1206 v3 — Content / State / Scene Rules Layer

Этот ZIP — **не полный Railway app** и не замена рабочей репы целиком.

Здесь лежат контентные, state и scene-rule слои для 1206:

- календарь сентября 1206;
- публичный и скрытый лор;
- Восточный сектор;
- NPC presence rules;
- `state/character_memory/` — динамическая память персонажей;
- `state/relationship_pairs/` — отношения парами;
- правила загрузки context/state;
- правила формата сцены, POV и нижнего блока;
- maintenance rules для мягкого recovery/cleanup.

## Важно

Не заливать этот ZIP поверх полноценной Railway-репы целиком, если в ней уже есть `app/`, ручные правки или актуальные карточки персонажей.

Безопасный способ интеграции:

1. Сначала сохранить/закоммитить текущую рабочую репу.
2. Подключать папки точечно.
3. Не заменять `characters/` целиком, если персонажи уже вручную очищены.
4. Не создавать новый слой правил поверх старого, если можно исправить файл-владелец.
5. Не использовать старый dynamic-state путь. Текущий стандарт: `state/character_memory/<id>.json`.

## Текущие стандарты

```yaml
characters/<id>/:
  main.yaml: паспорт персонажа
  character.yaml: характер, голос, поведение
  knowledge.yaml: стартовые знания/незнания
  past.yaml: прошлые события, если нужны

state/character_memory/<id>.json:
  динамическая память персонажа после сцен

state/relationship_pairs/<a>__<b>.json:
  динамика отношений пары

gpt/scene_output/:
  формат сцены, POV, footer, запреты качества

canon_lore/:
  общий лор, Восточный сектор, скрытый авторский лор

calendar/:
  календарь текущих сюжетных окон
```

## Главные правила

- Акира не грузится автоматически, если POV не Акира и её нет в сцене.
- Отношения не грузятся пачкой. Только нужные пары.
- Состояние и память персонажей пишутся в `state/character_memory`.
- Персонажи из `characters/` не являются фоновыми NPC.
- Hidden lore не становится знанием NPC без источника.
- Чистка state удаляет шум, но не protected moments.

## Character source rule

Персонажи должны читаться из полных карточек `characters/<id>/` и динамической памяти `state/character_memory/<id>.json`.

Не добавлять короткие отдельные summary-файлы вместо полных карточек: они делают поведение персонажей слишком плоским.
